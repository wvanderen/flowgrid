<objective>
Close the three diagnosed UAT gaps from `03-UAT.md` (status: diagnosed) that break or mar the protected first loop. These are regression/UX fixes rooted in real, confirmed root causes — not new features. The protected `open app → tap Cell → start session → finish → see rewards → return Home` loop must actually work end-to-end in the browser.

**Gap A (UAT test 15, severity MAJOR — simulation):** `complete_focus_session` appends the SessionRecord but never clears `cell.activeSessionStartedAt`. The existing `session-lifecycle.test.ts` covers start and cancel but never asserted the marker clears after complete, so the bug slipped through. `deriveActiveSession()` (dispatch.ts) therefore keeps projecting the session as active after Finish, and the GeneratorTile stays stuck in the "session in progress" state. User report: *"Finish triggers the session complete but session continues after pressing."*

**Gap B (UAT test 5, severity MINOR — UI/store):** `dispatch()` swallows rejected simulation results with a bare `return null` (the comment claims it surfaces issues but it does not), and `GeneratorTile` renders an enabled Start button for every non-active Cell even when another session is active elsewhere. The rejection is correct behavior (one-active-session invariant) but is invisible to the user. User report: *"It's rejected but the button just appears nonresponsive when clicked on the second cell."*

**Gap C (UAT test 15, severity MAJOR — UI):** The Cell Board happy path has no in-app navigation back to Flowgrid Home. Only the not-found branches have a Home link. User report: *"There's no way to return to the home view without hitting back or navigating via URL."*

**Gap D (UAT tests 2 & 15, severity COSMETIC — render):** `buildFlowgridScene` draws the Core at axial origin mapped directly to Pixi stage coordinates — pixel (0,0), the top-left corner. Ring Cells radiate from that top-left origin and clip against the canvas frame; some hexes become hard or impossible to tap. User report: *"Hexes aren't quite centered and some go off the edges of the frame."*

Output: a fully-fixed, buildable, fully-tested Phase 3 where the protected loop runs cleanly. The simulation fix (Gap A) carries a real regression test. No economy math, deterministic replay, accessibility semantics, or already-passing behavior changes.
</objective>

<root_causes>
## Confirmed root causes (do not re-investigate — these are verified)

**Gap A — `src/simulation/commands/complete-focus-session.ts:176-182`**
`cellAfterFocus` is built from `bloom.cell` (which descends from `advancedCell` ← `previousCell`) and overrides only `current` and `updatedAt`. It never sets `activeSessionStartedAt`. `cancel-focus-session.ts:75` correctly sets `activeSessionStartedAt: null`; complete-focus-session does not. The grep `rg "activeSessionStartedAt" src/simulation/commands/complete-focus-session.ts` returns zero matches — the field is simply never touched.

**Gap B — `src/app/store/dispatch.ts:71-76` + `src/ui/cell-board/GeneratorTile.tsx:46-66`**
The rejection branch reads `if (result.status !== 'applied') { return null; }`. The comment says it surfaces validation issues, but the code returns null without writing anything to the store. `GeneratorTile` renders its enabled Start button for every Cell where `!isThisCellActive`, regardless of whether `activeSession` is non-null on another Cell — so the user clicks, dispatch rejects, and nothing visible happens.

**Gap C — `src/ui/cell-board/CellBoard.tsx:87-116`**
The main render branch (Cell found) has no `<Link to="/">`. The two not-found branches (lines 57-64, 67-75) already have one. The happy path is missing the affordance.

**Gap D — `src/render/flowgrid/scene.ts:91-154`**
`buildFlowgridScene` computes `corePos = axialToPixel({q:0,r:0}, HEX_SIZE)` (= `{x:0, y:0}`) and draws everything into a `Container` that is added to `app.stage` with no position offset. Pixi stage origin is the top-left of the canvas, so the Core lands at the top-left corner and ring hexes radiate outward from there, clipping the frame.
</root_causes>
