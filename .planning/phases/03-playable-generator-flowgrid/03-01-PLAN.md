## Phase Goal (User Story)

**As a** Flowgrid user, **I want to** create Cells, start focus sessions from a Core-centered hex Flowgrid, and finish or cancel them safely while seeing Current, Bloom, Activation, and starter modules, **so that** the protected core loop of turning real effort into visible signal feels real and forgiving.

> Derived from ROADMAP Phase 3 Goal. This plan (03-01) delivers the **simulation truth layer** — the pure TypeScript game logic and durable-record extensions that Plans 03-02 (app shell + renderer) and 03-03 (Cell Board + session UI) consume. After this plan, all game logic for Cell CRUD, session lifecycle, daily rollover, Bloom Momentum, and Activation bonus exists, is tested, and is exactly replayable.

Extend the Phase 1/2 simulation with six new commands (create_cell, edit_cell, archive_cell, unarchive_cell, start_focus_session, cancel_focus_session), the Dexie v1→v2 migration (D-10), the day-rollover reconciliation system (D-13/D-14/D-16), the Bloom Momentum increment (D-14/SIM-06), and the Activation +% Current bonus in complete_focus_session (D-15/SIM-07).

Purpose: Every downstream UI plan depends on these types, commands, and systems existing and being correct. This plan proves the game logic in pure tests before any browser code lands.
Output: Extended domain records, six new command files, day-rollover system, Dexie v2 migration, and comprehensive simulation tests.
</objective>
