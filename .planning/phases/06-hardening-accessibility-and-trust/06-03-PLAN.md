## Phase Goal

**As a** Flowgrid user who relies on keyboard navigation or a screen reader, **I want to** reach and open every Cell without touching the canvas, **so that** the app is fully usable regardless of input method or WebGL availability.

## Objective

Deliver the accessible Cell-navigation vertical slice (UI-02): an always-visible semantic Cell list on FlowgridHome (D-06) that closes the one remaining UI-02 gap (opening an existing Cell was canvas-only). Every other critical action already has a semantic control from Phases 3–5; this plan adds the missing peer so the full critical-action set is keyboard/screen-reader reachable. After this plan, UI-02 is closed.

Purpose: D-06 is the literal fulfillment of the AGENTS.md accessibility rule ("canvas visuals must be paired with normal UI controls, semantic labels, and accessible panels"). The canvas stays `role="img"` (primary tap flow protected); the list is the accessible peer and doubles as the no-WebGL fallback for D-07 (Plan 06-02). This plan depends on 06-01 only because both edit FlowgridHome.tsx (the Settings header link from 06-01 must be in place first).

Output: Semantic Cell list in FlowgridHome.tsx, keyboard/semantic a11y component test.
