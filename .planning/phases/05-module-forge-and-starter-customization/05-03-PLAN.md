## Phase Goal

**As a** Flowgrid player **I want to** open the Forge from Home, see what my effort can buy me, pick a build choice, and see my modules get stronger — all through normal accessible UI **so that** the Forge feels like a meaningful global surface rather than a hidden mechanic.

## Objective

Plan 05-03 ships the user-facing half of Phase 5: the `/forge` route + `ForgePanel` + `ForgeSummary` + `ForgeChoiceList`, the `lastCompletedForge` store field + `captureCompletedForge` dispatch wiring, the ReturnCues Forge chip (D-12), and the extended Cell Board module tiles that show level badges + per-level effects (D-13, MOD-02).

This plan wires Plan 05-01's simulation truth and Plan 05-02's durable spine into the React surface. It mirrors Phase 4 wave-3 (`04-03-PLAN.md`): the `CorePanel` + `RejuvenationSummary` + `ReturnCues` chip pattern is the literal template. The protected `open app → tap Cell → start session` flow stays unobstructed: the Forge chip lives in the ReturnCues rail above the canvas and ForgeSummary is an inline panel (NOT a modal — D-11, Pitfall 7).

The plan has a non-blocking human verification checkpoint at the end because Phase 5's success criteria require a visual smoke (forge chip visible, choices render, summary persists) that the autonomous agent can prove is built but cannot fully confirm renders correctly in a real browser.

Purpose: Make the Forge a first-class user-facing surface and surface module levels in the existing inspector — closes MOD-02/MOD-03/MOD-06 at the UI layer.
Output: 3 new UI components + 1 new route + 5 modified files; `npm run build` + `npm test` green; the human smoke confirms the visual render.
