<objective>
This plan makes the Phase 4 economy playable in the browser. It adds the `/core` route with a six-piece Core panel (Energy, Charge, allocation, Integration progress, Module Tokens, next action), wires the live-timed rejuvenation lifecycle (start -> cosmetic timer -> finish/cancel) and the Activation-boost purchase into the dispatch path, surfaces a persisting inline RejuvenationSummary, and mounts a contextual return-cue stat-chip rail on FlowgridHome.

After this plan (plus the human visual smoke), a user can open Flowgrid, see stored Charge/Energy/tokens/near-Bloom as return cues, open /core, set allocation, start and finish a rejuvenation that visibly processes Charge into Integration and grants Module Tokens, and spend Energy on the Activation boost — completing the activity/rest alternation loop.

Purpose: The browser-playable vertical slice on top of the simulation truth (04-01) and durable spine (04-02). UI dispatches commands and reads selectors only; it never computes economy rules (boundary-enforced).
Output: CorePanel + RejuvenationSummary + RejuvenationTimer + nextCoreAction + ReturnCues + /core route + store/dispatch extensions.
</objective>
