<objective>
Plan 01 builds the **persistent canvas spine** — the load-bearing MVP slice that dissolves the structural blocker from 06-05 Task 3. It lifts `FlowgridCanvas` out of the `/` route swap into a pathless React Router v7 layout route so the Pixi Application + scene are created ONCE and survive all navigation across `/`, `/cells/:id`, `/core`, with `/settings` and `/forge` rendered as full-screen overlays ABOVE the still-mounted canvas. After this plan, every particle-emitting event (active-session Current trails, finish Bloom/Core ripple, Core convert/store, rejuvenation flash) fires with the canvas on-screen.

This plan implements D-01 (URL reflects view; canvas persists in layout slot), D-02 (takeovers are overlays above the mounted canvas + ticker pause), D-03 (home chrome persists across core views), and D-05 (no /history route — policy only). It preserves the Phase 6 D-05 build-once invariant as the spine of the pivot.

Purpose: Make effort perceivable as it happens by keeping the canvas mounted during the events that emit its signal.
Output: AppLayout component, restructured routes, two new store view-state fields, takeover overlay wrappers, slimmed HomeDock, extended layout-persistence UI test.
</objective>
