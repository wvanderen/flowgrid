## Tests

### 1. Cold Start Smoke Test
expected: Run `npm run dev` from a clean state. App boots without errors. Home route (/) loads with the Flowgrid canvas visible (a <canvas> rendering the hex lattice). No console errors about Pixi/WebGL init. Canvas immediately visible.
result: pass

### 2. Canvas Persists When Selecting a Cell
expected: From home (/), tap a Cell (or click a Cell in the Cell-list nav). You land on /cells/:id. The SAME canvas stays mounted and visible — it does NOT rebuild, flicker, or go blank during navigation. The hex grid remains on screen the whole time.
result: pass

### 3. Z-Lift Spotlight on Selected Cell
expected: On /cells/:id with a Cell selected, the selected Cell's hex visibly lifts (scales up ~1.3x) and non-selected neighbor hexes dim (lower opacity). A spotlight/focus cone visual highlights the selected Cell. The container framing does NOT move — only the selected hex scales.
result: pass

### 4. ZLiftDock Controls Beside Canvas
expected: On /cells/:id, a semantic control dock appears beside the canvas showing Start (Focus Session) / Finish / Cancel buttons (and the Cell name). The dock is real HTML (not drawn on canvas). Buttons are keyboard-accessible. The same controls exist whether on desktop (side dock) or narrow viewport (bottom strip).
result: pass

### 5. Start Focus Session From the Dock
expected: Click Start in the dock (or tap the Cell's hex). A focus session begins. The canvas stays visible and mounted throughout — the session does NOT navigate away or cover the canvas. Session state is reflected inline.
result: pass

### 6. Finish Session Shows Live Signal
expected: Finish the focus session. You see effort become visible signal in real time on the canvas — particles/Current flowing, a ripple, or visual feedback as the Core convert/store happens. The canvas does NOT go blank or rebuild during finish. (This is the re-opened 06-05 Task 3 human smoke — perceptual confirmation that particles are visible during events.)
result: pass

### 7. Canvas Persists on Core Route
expected: Navigate to /core. The SAME canvas stays mounted and visible — no rebuild or flicker. Core convert/store interactions happen with the canvas visible alongside. The Cell-list nav also stays present on /core.
result: pass

### 8. Settings Opens as Full-Screen Takeover
expected: Navigate to /settings. A full-screen overlay covers the canvas (Settings panel fills the screen). The canvas stays MOUNTED underneath (not destroyed) — and its motion/ticker is paused while covered. Navigate back: the canvas is instantly visible again with NO rebuild/re-init delay.
result: pass

### 9. Forge Opens as Full-Screen Takeover
expected: Navigate to /forge. A full-screen overlay covers the canvas (Forge build choices fill the screen). Same as Settings: canvas stays mounted underneath, motion paused. Navigate back: canvas instantly visible, no rebuild.
result: pass

### 10. Edit Cell Dialog Opens Above Canvas
expected: On /cells/:id, open the Edit Cell dialog (Edit button in Cell actions). The dialog renders ABOVE the canvas (Radix Portal escapes the stacking context) — it's not hidden behind the canvas. Edit a field and submit; the dialog closes and dispatches the edit. Closing without submitting does NOT dispatch.
result: pass

### 11. Mobile Cell-List Collapses to Menu
expected: Resize the viewport narrow (below the md: breakpoint, e.g. ~375px wide). The always-visible Cell-list nav collapses into a menu button (Radix Menu Cell-switcher). Click it to open the menu; arrow keys navigate the Cell options; selecting one navigates to that Cell. At desktop width the always-visible nav returns.
result: pass
