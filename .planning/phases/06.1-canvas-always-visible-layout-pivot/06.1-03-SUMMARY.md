## Decisions Made

- **SPA navigation over `page.goto` for the Pitfall 1 test:** `page.goto('/cells/B')` triggers a full page reload that ALWAYS destroys the canvas regardless of the layout route — it cannot verify Pitfall 1 (which is specifically about React Router's in-app navigation keeping the layout mounted). The Cell-list `<nav>` is persistent chrome at desktop width (Playwright's default 1280×720 viewport hits the `md:` breakpoint), so clicking the Param Cell B link from `/cells/A` exercises a TRUE param-only child swap.
- **Scoped CellBoard heading + Start/Finish to the CellBoard region:** Plan 02's ZLiftDock renders `<h2>{cellName}</h2>` and Start/Finish/Cancel buttons with the SAME accessible names as CellBoard's (two-paths-one-truth — D-08). This creates Playwright strict-mode ambiguity. Rule 3 fix: scope to `level:1` for headings and to `getByRole('region', { name: 'Cell Board for X' })` for buttons. This was applied to BOTH my new Test D AND the existing `runSessionAndReadSummary` helper (which the existing reduced-motion durability test depends on).
- **ZLiftDock stubbed as null in cell-board.test.tsx:** leaving it real would have caused Start/Finish button ambiguity in the EXISTING cell-board tests (which use `getByRole('button', { name: 'Start Focus Session' })` without scoping). ZLiftDock's dispatch parity is independently covered by `tests/ui/z-lift-dock.test.tsx`, so stubbing it keeps CellBoard the unambiguous system under test.
- **D-09 WebGL-fail continuity asserts the mock contract, not the real component:** the test verifies the mock's fail branch mirrors the real FlowgridCanvas fail branch (same className sizing + role=status). The real component's actual no-reflow behavior is exercised by the Playwright suite in a real browser. The unit test pins the contract so future mock drift is caught.
- **Used `vi.hoisted` for the per-test mock branch:** vitest hoists `vi.mock` calls to the top of the file, so the mock factory cannot reference a top-level `let` declared after it. `vi.hoisted` creates a hoisted mutable binding the mock factory can close over safely; the beforeEach resets it so test order doesn't leak mock state.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] CellBoard vs ZLiftDock accessible-name collisions broke the existing `runSessionAndReadSummary` helper and the new Test D**
- **Found during:** Task 1 (canvas-smoke E2E first run)
- **Issue:** Plan 02's ZLiftDock renders `<h2>{cellName}</h2>` and Start/Finish/Cancel buttons with `aria-label="Start Focus Session"` / `"Finish"` / `"Cancel"` — identical accessible names to CellBoard's `<h1>{cellName}</h1>` and Start/Finish/Cancel buttons (two-paths-one-truth is a Plan 02 feature). This caused Playwright strict-mode violations: `getByRole('heading', { name })` resolved to 2 elements; `getByRole('button', { name: 'Start Focus Session' })` resolved to 2 elements. The violation affected BOTH my new Test D AND the pre-existing `VER-06 (reduced-motion durability)` test (which the existing helper drives) — Plan 02 introduced the regression but it only surfaced when Plan 03 extended the canvas-smoke suite.
- **Fix:** Scope heading lookups to `level: 1` (CellBoard's `<h1>`); scope Start/Finish/Cancel button clicks to `page.getByLabel('Cell Board for X')` (CellBoard's `<section aria-label>`). Applied to the helper (1 site) and to Test D (1 site).
- **Files modified:** tests/e2e/canvas-smoke.spec.ts
- **Verification:** `npx playwright test canvas-smoke` → 6 passed, 1 expected skip.
- **Committed in:** c26c5ad (Task 1 commit — fix bundled with the new tests).

**2. [Rule 3 - Blocking] Takeover-coverage test intermittently failed: canvas count 0 immediately after `page.goto('/settings')`**
- **Found during:** Task 3 automated-prerequisites re-verification (after Task 2)
- **Issue:** `page.goto('/settings')` triggers a full page reload; the SettingsTakeover dialog renders quickly but the Pixi v8 Application mounts asynchronously. An instant `page.locator('canvas').count()` immediately after `expect(dialog).toBeVisible()` raced the mount and returned 0 about 1 in 3 runs.
- **Fix:** Added `await page.locator('canvas').waitFor({ state: 'attached' })` before the count assertion so it retries until the canvas is in the DOM.
- **Files modified:** tests/e2e/canvas-smoke.spec.ts
- **Verification:** 3 consecutive reliable runs of the takeover test (was ~1-in-3 flaky).
- **Committed in:** 45dbcba (Task 1 follow-up fix).

**3. [Rule 3 - Blocking] Used SPA navigation (not `page.goto`) for the param-change identity test**
- **Found during:** Task 1 first run (Test B failed: "canvas DOM identity must persist" received false)
- **Issue:** My initial Test B used `page.goto('/cells/A')` then `page.goto('/cells/B')`. `page.goto` always triggers a full page reload that destroys the canvas regardless of the layout route — so the test was meaningless (it would always fail, falsely suggesting Pitfall 1 is open). The test's intent is to verify React Router's in-app navigation keeps the layout mounted across a param-only child swap.
- **Fix:** Use the Cell-list `<nav>` persistent chrome (visible at desktop width) to navigate via SPA link clicks: `/` → click "Param Cell A" → `/cells/A` → tag canvas → click "Param Cell B" → `/cells/B` → verify tagged canvas is still `.isConnected`.
- **Files modified:** tests/e2e/canvas-smoke.spec.ts
- **Verification:** Test B now passes — the SAME canvas DOM node survives the param-only SPA navigation (Pitfall 1 truly closed).
- **Committed in:** c26c5ad (Task 1 commit — correct navigation in the initial test).

**4. [Rule 3 - Blocking] `screen.getByLabel` is not a RTL API — used `getByRole('region', { name })` for CellBoard's `<section aria-label>`**
- **Found during:** Task 2 first run (cell-board layout-route tests failed: "screen.getByLabel is not a function")
- **Issue:** Playwright's `page.getByLabel(...)` works for any aria-label, but RTL's API is role-based. For CellBoard's `<section aria-label="Cell Board for X">`, the correct RTL query is `screen.getByRole('region', { name: ... })` (a `<section>` with an accessible name resolves to the `region` role).
- **Fix:** Replaced `screen.getByLabel(...)` with `screen.getByRole('region', { name: ... })`; used `within(cellBoardSection).getByRole('link', ...)` for scoped link lookups.
- **Files modified:** tests/ui/cell-board.test.tsx
- **Verification:** `npx vitest run tests/ui/` → 61 passed.
- **Committed in:** 1d780ce (Task 2 commit).
